. Rules先行，减少幻觉
•	Thiết lập 5–10 quy tắc rõ ràng cho dự án trong file .cursorrules ở thư mục gốc.
•	Quy 1tắc giúp Cursor hiểu cấu trúc và giới hạn của dự án, giảm lỗi "hallucination".
•	Với dự án có sẵn, dùng lệnh /gen để tự động sinh quy tắc qua Generate Cursor Rules.
•	Tham khảo các bộ quy tắc cộng đồng: 
o	awesome-cursorrules
o	cursor-list
o	cursor-directory
________________________________________
2. 精准提示，代码无忧
•	Tránh nhập prompt mơ hồ → kết quả code dễ sai hoặc không phù hợp.
•	Cấu trúc prompt chuẩn: Công nghệ + Yêu cầu hành vi + Ràng buộc điều kiện.
•	Ví dụ:
“Dùng React + TypeScript viết component OAuth2.0 login, hỗ trợ username/password, xử lý token sau khi login thành công, tương thích trình duyệt, không dùng thư viện ngoài, nút hỗ trợ dark mode.”
•	Mẹo: chuyển prompt thô sang dạng có cấu trúc trong Ask mode, rồi chạy ở Agent mode.
________________________________________
3. 逐文件工作，分治高效
•	Áp dụng chiến lược chia nhỏ: xử lý từng file một theo quy trình: tạo → kiểm thử → xem lại.
•	Giúp tập trung, tránh loạn khi làm toàn bộ project.
________________________________________
4. 测试先行，质量保障
•	Áp dụng TDD (Test-Driven Development): viết test case trước → bảo Cursor sinh code cho đến khi test pass.
•	Phát hiện lỗi sớm, giảm chi phí sửa lỗi sau này.
________________________________________
5. 人工审查，告诉AI以此为鉴
•	Luôn kiểm tra lại code do AI sinh ra → phát hiện lỗi logic, cú pháp, hoặc không đúng yêu cầu.
•	Khi sửa lỗi, hãy bảo Cursor tạo quy tắc mới từ lỗi đó và thêm vào Project/User Rule (tùy phạm vi áp dụng).
________________________________________
6. 精准聚焦，防止误伤
•	Dùng @file, @folder, @git để giới hạn phạm vi chỉnh sửa.
•	Tránh ảnh hưởng tới module không liên quan khi làm việc trong dự án lớn.
________________________________________
7. 文档加持，全局掌控
•	Đặt tài liệu thiết kế & checklist vào thư mục .cursor/ → giúp AI hiểu rõ ngữ cảnh dự án.

•	Cursor học nhanh hơn từ hành động sửa thực tế.
•	Thêm comment rõ ràng và lưu file sau khi sửa.
